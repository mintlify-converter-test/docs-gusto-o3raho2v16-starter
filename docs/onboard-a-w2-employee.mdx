title: "Employees in Payroll"
description: "Overview of employee management in payroll"
---
Employees are a critical part of payroll. Companies themselves are, naturally, a collection of one to hundreds of employees that must be paid on a regular cadence for work done.

Employees may be added during a company’s onboarding for payroll, or at any time after the initial onboarding is complete. An employee can be setup in one of two ways:

1.  By Employer
2.  By Employee (“Self-onboarding”)

Both can be done using Gusto’s pre-built Flows or using the Employee APIs to build custom UI/UX.

To learn more about the difference between Employee Onboarding by Employer vs Employee Self-Onboarding review the [Onboarding Method Comparison Guide](/embedded-payroll/docs/onboarding-method-comparison)
.

#### The steps for both of these processes is the same

## 1\. Create a W2 Employee

You can create an employee using the `POST companies/{company_id}/employees` [endpoint](/embedded-payroll/reference/post-v1-employees)
.

The example below shows an employee "self-onboarding".

```curl
curl --request POST \
     --url https://api.gusto-demo.com/v1/companies/{company_uuid}/employees \
     --header 'accept: application/json' \
     --header 'authorization: Bearer COMPANY_API_TOKEN' \
     --header 'content-type: application/json' \
     --data '
{
     "first_name": "Alexander",
     "middle_initial": "A",
     "last_name": "Hamilton",
     "date_of_birth": "1979-06-01",
     "email": "sonofanorphan@example.com",
     "ssn": "123451776",
     "self_onboarding": true
}
'

``````curl
const fetch = require('node-fetch');

const url = 'https://api.gusto-demo.com/v1/companies/{company_uuid}/employees';
const options = {
  method: 'POST',
  headers: {
    accept: 'application/json',
    'content-type': 'application/json',
    authorization: 'Bearer COMPANY_API_TOKEN'
  },
  body: JSON.stringify({
    first_name: 'Alexander',
    middle_initial: 'A',
    last_name: 'Hamilton',
    date_of_birth: '1979-06-01',
    email: 'sonofanorphan@example.com',
    ssn: '123451776',
    self_onboarding: true
  })
};

fetch(url, options)
  .then(res => res.json())
  .then(json => console.log(json))
  .catch(err => console.error('error:' + err));

```

### b. Update an Employee's Federal Taxes

To update the Federal Tax Details, you will first need to get the `version` of the tax details using [`GET employees/{employee_uuid}/federal_taxes`](/embedded-payroll/reference/get-v1-employees-employee_id-federal_taxes)
 . Once you have the `version` you can update the federal tax details using [`PUT employees/{employee_uuid}/federal_taxes`](/embedded-payroll/reference/put-v1-employees-employee_id-federal_taxes)

### d. Create an Employee Bank Account

Creating an employee bank account will automatically update an employee’s payment method from `Check` to `Direct Deposit`. If you want to change the payment method you can use the `PUT employees/{employee_uuid}/payment_method` [endpoint](/embedded-payroll/reference/put-v1-employees-employee_id-payment_method)
. To create a bank account for an Employee use the `POST employees/{employee_uuid}/bank_account`[endpoint](/embedded-payroll/reference/post-v1-employees-employee_id-bank_accounts)
.

```curl
curl --request POST \
     --url https://api.gusto-demo.com/v1/employees/{employee_uuid}/bank_accounts \
     --header 'accept: application/json' \
     --header 'authorization: Bearer COMPANY_API_TOKEN' \
     --header 'content-type: application/json' \
     --data '
{
     "name": "BoA Checking Account",
     "routing_number": "266905059",
     "account_number": "5809431207",
     "account_type": "Checking"
}
'

``````curl

## 6\. Mark Employee as Onboarded

With the basic details, address, payment method, and tax information complete, the final step is to mark the employer as `onboarding_completed`. This is done using the `PUT employees/{employee_uuid}/onboarding_status` [endpoint](/embedded-payroll/reference/put-v1-employees-employee_id-onboarding_status)
. You will want to change the status to `onboarding_completed` in the body.

```curl
curl --request PUT \
     --url https://api.gusto-demo.com/v1/employees/employee_id/onboarding_status \
     --header 'accept: application/json' \
     --header 'authorization: Bearer COMPANY_API_TOKEN' \
     --header 'content-type: application/json' \
     --data '
{
     "onboarding_status": "onboarding_completed"
}
'

``````curl
const fetch = require('node-fetch');

const url = 'https://api.gusto-demo.com/v1/employees/employee_id/onboarding_status';
const options = {
  method: 'PUT',
  headers: {
    accept: 'application/json',
    'content-type': 'application/json',
    authorization: 'Bearer COMPANY_API_TOKEN'
  },
  body: JSON.stringify({onboarding_status: 'onboarding_completed'})
};

fetch(url, options)
  .then(res => res.json())
  .then(json => console.log(json))
  .catch(err => console.error('error:' + err));

```
