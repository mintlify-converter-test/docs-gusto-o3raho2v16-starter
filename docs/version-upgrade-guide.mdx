## Authentication and Scope Changes

| Version | Breaking Change | Update Required |
| --- | --- | --- |
| [v2023-12-01](https://docs.gusto.com/embedded-payroll/changelog/v2023-12-01) | The [Get the current user endpoint (v1/me)](https://docs.gusto.com/embedded-payroll/v2023-09-01/reference/get-v1-me) has been deprecated. | Migrate to the [Get info about the current access token](https://docs.gusto.com/embedded-payroll/reference/get-v1-token-info) |
| [v2023-05-01](/embedded-payroll/changelog/v2023-05-01) | Starting from version 2023-05-01, all endpoints that authenticate with an access token require a strict access token. A strict access token is reserved for access to only a single company. Requests using tokens that do not meet this requirement shall be responded with a forbidden (403) status. | See [Strict Access guide](/embedded-payroll/docs/strict-access) |
| [v2023-03-01](/embedded-payroll/changelog/v2023-03-01) | The following endpoints require additional scopes, which are specified in the [changelog](/embedded-payroll/changelog/v2023-03-01):  [Get the current user](/embedded-payroll/reference/get-v1-me)  [Get a company](/embedded-payroll/reference/get-v1-companies)  [Get all payrolls for a company](/embedded-payroll/reference/get-v1-companies-company_id-payrolls)  [Get a single payroll](/embedded-payroll/reference/get-v1-companies-company_id-payrolls-payroll_id)  [Get a company benefit](/embedded-payroll/reference/get-v1-company_benefits-company_benefit_id) | No changes required. Reach out to your Technical Solutions representative for any issues. |

## Payroll

| Version | Breaking Change | Update Required |
| --- | --- | --- |
| [v2023-09-01](/embedded-payroll/changelog/v2023-09-01) | Changes to the [payroll blockers endpoint](/embedded-payroll/reference/get-v1-companies-payroll-blockers-company_uuid):  1\. `needs_onboarding` has been deprecated as a blocker type  2\. Added new blocker types `pending_information_request` and `pending_recovery_case`, which were previously folded into `pending_payroll_review` | 1\. Leverage our [more granular onboarding blockers](/embedded-payroll/docs/payroll-blockers)  2\. Monitor for the `pending_information_request` or `pending_recover_case blockers` for open information requests or recovery cases |

Added `processing_statuses` param used to return `processed`, `unprocessed` or both types of payrolls, defaults to `processed`  5\. Added `payroll_types` param used to return `regular`, `off_cycle`, or both types of payrolls, defaults to `regular` | 1\. Optionally request totals with the include param, i.e. `?include=totals`  2\. Depends on partner implementation  4\. To retrieve both processed and unprocessed payroll, use `processing_statuses=unprocessed,processed`  5\. To retrieve regular and off-cycle payroll, use `payrolls_types=regular,off_cycle` |
| [v2023-04-01](/embedded-payroll/changelog/v2023-04-01) | Changes to the [get a single payroll endpoint](/embedded-payroll/reference/get-v1-companies-company_id-payrolls-payroll_id)  1\. `show_calculation` parameter has been removed  2\. `version` attribute has been removed  3\. `employee_compensations` and `totals` attributes have been removed for unprocessed payrolls | 1\. | [v2022-11-01](/embedded-payroll/changelog/v2022-11-01) | `payroll_id` is being deprecated in favor of `payroll_uuid`.
