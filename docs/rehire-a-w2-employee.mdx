Gusto provides the capability to rehire a previously dismissed employee to facilitate quick and easy re-onboarding for payroll using the Rehire Flow or API. The Rehire Flow includes the ability to review an employeeâ€™s onboarding.

## 1\. Create a Rehire

To create a rehire use the `POST employees/{employee_uuid}/rehire` [endpoint](post-v1-employees-employee_id-rehire)
.

The following path params are required:

*   **employee\_uuid** - the uuid of the employee for which youâ€™d like to rehire

The following body params are required:

*   **effective\_date** - the date of the employee's rehire
*   **work\_location\_uuid** - the uuid of the location where the employee is employed. This can be acquired by fetching the company locations.
*   **file\_new\_hire\_report** - elect to file a new hire report for a given employee's work state

```curl
curl --request POST  
     --url <https://api.gusto-demo.com/v1/employees/{employee_uuid}/rehire>  
     --header 'accept: application/json'  
     --header 'authorization: Bearer {{access_token}}' \  
     --header 'content-type: application/json'  \
     --data '  
{  
     "effective_date": "2023-01-01",  
     "work_location_uuid": â€œb125fe86-981e-47a2-b7a2-58ca8de1dcb9â€,  
     "file_new_hire_report": true,  
}'

``````curl

## 2\. Update a Rehire

.

The following path params are required:

*   **employee\_uuid** - the uuid of the employee for which youâ€™d like to rehire

The following body params are required:

*   **version** - the rehire's current version identifier. This can be acquired by fetching the rehire.
*   **effective\_date** - the date of the employee's rehire

## 4\. Cancel a Rehire

To cancel a rehire use the `DELETE employees/{employee_uuid}/rehire` [endpoint](delete-v1-employees-employee_id-rehire)
.

The following path params are required:

*   **employee\_uuid** - the uuid of the employee

```curl

     --url <https://api.gusto-demo.com/v1/employees/{employee_uuid}/rehire>  
     --header 'accept: application/json'  
     --header 'authorization: Bearer {{access_token}}' \  
     --header 'content-type: application/json'  \

``````curl
const fetch = require('node-fetch');

const options = {
	method: 'DELETE',
	headers: {
		accept: 'application/json',
		'content-type': 'application/json',
		authorization: 'Bearer COMPANY_API_TOKEN'
	}

fetch(url, options)
	.then(res => res.json())

### 1\. Enter employee rehire details

Enter the rehire start date, work address, and whether you want Gusto to file a new hire report on your behalf (this is the same option as offered when an employee is first onboarded, and the user should consult state requirements as to whether this is needed).

If the company is taxable as S-corp, the 2% shareholder field to select whether the employee is a 2% shareholder will be shown.
