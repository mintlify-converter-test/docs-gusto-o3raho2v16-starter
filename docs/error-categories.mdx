---
title: "Error Categories"
description: "List of error categories for integrations"
---
The following error samples provide a comprehensive list of all error categories we currently support. You may use the category to programmatically handle errors in your integration.

### deprecated\_endpoint

```json
404 Not Found

{
  "errors": [
    {
      "error_key": "request",
      "category": "deprecated_endpoint",
      "message": "The requested endpoint is no longer supported in the requested API version."
    }
  ]
}

```

### duplicate\_operation

```json
422 Unprocessable Entity

{
  "errors": [
    {
      "error_key": "base",
      "category": "duplicate_operation",
      "message": "Cannot have multiple of the same state field per company"
    }
  ]
}

```

### internal\_error

If an internal error occurs, our engineers are notified to investigate the problem.

```json
500 Internal Server Error

{
  "errors": [
    {
      "error_key": "request",
      "category": "internal_error",
      "message": "An unexpected error has occurred."
    }
  ]
}

```

### invalid\_api\_version

To learn more about API Versions review our [API Versioning guide](/embedded-payroll/docs/api-versioning)
.

```json
406 Not Acceptable

{
  "errors": [
    {
      "error_key": "request",
      "category": "invalid_api_version",
      "message": "Invalid API Version `2023`. Expected format: YYYY-MM-DD"\
    }\
  ]
}

```
